  ```typescript
  import { useQuery } from '@tanstack/react-query';
  const { data } = useQuery({
    queryKey: ['ai_job', jobId],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('ai_jobs')
        .select('status, last_error')
        .eq('id', jobId!)
        .single();
      if (error) throw error;
      return data;
    },
    enabled: !!jobId,
    refetchInterval: (query) => {
      const status = query.state.data?.status;
      if (status === 'done' || status === 'failed') return false;
      return 3000; // 3 seconds
    },
    staleTime: 0,
  });
  ```

  ```typescript
  const SVG_PADDING = 32;
  const screenWidth = Dimensions.get('window').width - SVG_PADDING * 2;
  const scale = screenWidth / dimensions.widthM;
  const svgWidth = screenWidth;
  const svgHeight = dimensions.heightM * scale;
  ```
- Render order:
  1. Background Rect: full size, fill `#F5F0E8`
  2. Garden boundary: Rect with stroke `#8B7355`, strokeWidth 2, strokeDasharray "6 3" (organic/dashed feel), no fill, strokeLinecap="round", strokeLinejoin="round"
  3. Grid (if showGrid): horizontal + vertical Lines at 1m intervals. Stroke `#D6CFC4`, strokeWidth 0.5, opacity 0.4
  4. Elements: for each element, render positioned Rect (or Circle for trees) with:
     - x/y = element.xM - element.widthM/2 (center-based positioning to top-left)
     - width/height = element.widthM / element.heightM (in SVG units = meters * scale)
     - fill = PLAN_COLORS[element.elementType] ?? PLAN_COLORS.Sonstiges
     - opacity 0.7 (sketch-like transparency per rendering rules)
     - strokeWidth 1.5, stroke = darker variant of fill, strokeLinecap="round", strokeLinejoin="round"
     - Special: Baum → Circle instead of Rect (radius = widthM/2)
     - Special: Zaun → Line with strokeDasharray (no fill)
  5. Element labels: SvgText centered on each element, 14pt equivalent (fontSize scaled), fill `#8B7355`, truncated if > 10 chars
  6. Dimension labels: SvgText along boundary edges showing meter values (e.g. "12 m" along bottom, "8 m" along left side)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase DB -> Client (ai_results) | Parsed AI results consumed by client UI for rendering |
| Client -> react-native-svg | Element data rendered as SVG; no code execution risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-04-01 | Tampering | Malicious parsed_result from ai_results | mitigate | parseElements (Plan 02) already validates/coerces all fields. Client uses typed PlanElementCandidate with Number.isFinite checks. Malformed data results in missing elements, not crashes. |
| T-4-04-02 | Denial of Service | Infinite polling loop | mitigate | TanStack Query refetchInterval returns false when status='done'|'failed'. Client-side 120s timeout kills polling. |
| T-4-04-03 | Information Disclosure | ai_results leaked cross-garden | mitigate | ai_results has RLS (from Phase 1 migration). Client queries with garden_id filter. Non-members get 0 rows. |
| T-4-04-04 | Spoofing | Unauthenticated SVG rendering | accept | SVG rendering is client-only from local data. No security boundary crossed. Plan data already RLS-protected at fetch time. |
</threat_model>
