## Interaction Contract

### Capture Flow Navigation (D-01, D-02, D-05)

- Linear wizard flow: Step 1 (Overview) -> Step 2 (North) -> Step 3 (South) -> Photo Review -> Dimensions -> Analysis -> Confirmation -> Plan View
- Back navigation: hardware/gesture back returns to previous step. No data loss -- captured photos persist in local state across steps.
- Step indicator: 3 dots at top of each capture screen. Active dot: accent green, 8px diameter. Inactive: stone-300, 8px. Completed: accent green filled, 8px. Horizontal layout, centered, 8px gap between dots.
- Skip forward: NOT allowed. Steps are sequential. User must capture or select a photo for each step before proceeding.
- Each step: camera viewfinder fills screen. Bottom overlay (semi-transparent dark, 120px height) contains: instruction text + example image + "Foto aufnehmen" shutter button (center, 64px diameter circle, accent border 3px, white fill) + "Aus Galerie" text button (below shutter, stone-300 text, 14pt).

### Photo Review Screen (D-03)

- Grid layout: 3 columns, 8px gap. Each photo as PhotoThumbnail (80x80px minimum, aspect-fill crop).
- Photos are numbered (1, 2, 3, ...) with a small badge in the top-left corner of each thumbnail.
- "Weiteres Foto hinzufuegen" button: dashed border card (stone-300, 80x80px) with Plus icon (24px, accent green) centered. Tapping opens camera/gallery picker (no angle guidance for extra photos).
- Footer: "Weiter zu Gartenform" primary CTA button, full width, accent green. Disabled state (stone-300 bg, stone-400 text) if fewer than 1 photo captured.
- 1-photo warning (PHOTO-07, D-08): if exactly 1 photo, show InlineBanner above CTA: "Nur 1 Foto vorhanden. Mehr Perspektiven verbessern die Erkennung. Analyse wird trotzdem versucht." Variant: warning (amber). Dismissible. Does not block proceeding.

### Shape Selection + Dimensions (D-04)

- Top section: "Grundform waehlen" heading (20pt semibold).
- ShapeSelector: 2x2 grid of shape cards. Default selection: Rechteck (most common).
- Below shape grid: dynamic dimension input fields appear based on selected shape:
  - **Rechteck:** 2 fields -- "Laenge" + "Breite" (both DimensionInput with "m" suffix)
  - **L-Form:** 4 fields -- "Laenge gesamt", "Breite gesamt", "Einschnitt Laenge", "Einschnitt Breite"
  - **Trapez:** 3 fields -- "Seite oben", "Seite unten", "Tiefe"
  - **Freihand:** Instructions text "Tippe auf die Karte, um Eckpunkte zu setzen" + interactive point-placement area (200px height, stone-100 background, tap to add points, long-press to remove last point). Meter input per edge auto-calculated or manually editable.
- Input validation: all dimensions > 0 and <= 200m. Error text below field in red-600, 14pt.
- CTA: "Analyse starten" primary button at bottom. Disabled until at least one valid dimension pair exists.

### Analysis Loading (NFR-02, NFR-03)

- Full-screen overlay (same pattern as ExtractionLoader from Phase 2).
- Centered content: Loader2 icon (48px, accent green, spinning) + heading "Garten wird analysiert..." (20pt semibold) + sub-text "Das kann 30-60 Sekunden dauern." (14pt, stone-500) + indeterminate progress bar (accent green, full width, animate-pulse).
- Cancel button at bottom: "Analyse abbrechen" outline button. Cancels the analysis, returns to dimensions screen.
- Budget soft-warning (NFR-03): if user has >= 50 calls today, show BudgetWarningBanner above the analysis trigger (on the dimensions screen, before overlay appears). User can still proceed.
- Budget hard-stop (NFR-03): if user has >= 200 calls today, CTA "Analyse starten" is disabled. BudgetWarningBanner shows destructive variant: "Tageslimit erreicht. Weitere Analysen sind morgen wieder moeglich."
- Timeout: 120s client-side. On timeout: overlay transitions to error state -- AlertCircle icon (48px, red-600) + error heading + "Erneut versuchen" primary button + "Analyse abbrechen" outline button.
- On success: auto-navigate to Element-Bestaetigung screen. No manual "Continue" needed.

### Element Confirmation (D-06, PHOTO-05)

- Screen title: "Elemente bestaetigen" (20pt heading).
- Sub-text: "{n} Elemente erkannt" (14pt label, stone-500).
- "Alle bestaetigen" shortcut button: secondary button (outline variant) at top of list. Toggles all elements to accepted state. If all already accepted, label changes to "Alle abwaehlen".
- Scrollable list of PlanElementRow components. Each row:
  - Left: element type icon (24px, color-coded per plan palette)
  - Center: element name (16pt body, e.g. "Rasenflaeche", "Gartenlaube", "Kompostplatz") + estimated size label below (14pt label, stone-500, e.g. "ca. 3 x 2 m")
  - Right: ConfidenceBadge ("sicher" / "unsicher") + Switch (accept/reject)
  - Row height: min 52px, padding 16px vertical / 16px horizontal
  - Accepted: full opacity, switch on (accent green track)
  - Rejected: stone-400 text, switch off (stone-300 track)
- Footer: "Plan erstellen" primary CTA, full width, accent green. Always enabled (0 accepted elements = empty plan, per PHOTO-08).
- Zero elements (PHOTO-08, D-07): list replaced with empty state. Friendly illustration placeholder (stone-200 circle, 80px, with seedling icon). Heading: "Keine Elemente erkannt" (20pt). Body: "Kein Problem -- du kannst im Editor alles manuell einzeichnen." (14pt, stone-500). CTA: "Leeren Plan erstellen" (accent green button).

### Plan View (D-09, PHOTO-06)

- Screen title: "Dein Gartenplan" (20pt heading).
- Plan rendered as SVG inside a ScrollView (pinch-to-zoom deferred to Phase 5; simple ScrollView horizontal/vertical scroll for now).
- Plan container: full width minus 32px horizontal padding. Aspect ratio derived from garden dimensions. Background: `#F5F0E8` (paper beige).
- Grid toggle: icon button top-right (Grid3x3 icon from lucide, 24px, `accessibilityLabel="Raster ein/aus"`). Toggles 1m grid overlay on/off. Default: ON.
- Element labels: small text labels (14pt, `#8B7355`) centered on each element shape. Truncated with ellipsis if > 10 characters.
- Dimension labels: meter values along garden boundary edges (14pt, `#8B7355`, positioned outside boundary line).
- No tap interaction on plan elements in Phase 4 (interactivity deferred to Phase 5 to control scope per Claude's Discretion area decision).
- Footer: "Zum Editor" primary CTA navigates to Phase 5 entry (or placeholder if Phase 5 not yet built). Below CTA: "Erneut erfassen" text button (stone-500, 14pt) to restart the capture flow.
- Plan data is persisted to `plan_elements` + `garden_dimensions` tables via writeWithOutbox before this screen renders.

## States per Screen

### Capture Step Screens (Step 1/2/3)
- Camera ready: viewfinder live, shutter button active (accent border, white fill)
- Camera permission denied: full-screen message "Kamerazugriff benoetigt" + "Einstellungen oeffnen" button + "Aus Galerie waehlen" fallback button
- Photo captured: brief flash animation (white overlay, 100ms fade), auto-advance to next step after 300ms delay
- Gallery selected: photo loaded into preview, "Weiter" button appears

### Photo Review Screen
- Default (3 photos): grid with 3 thumbnails + add-more card
- 1 photo only: grid with 1 thumbnail + add-more card + InlineBanner warning below grid
- 0 photos: should not be reachable (flow requires minimum 1 step completion)
- Extra photos (4+): grid wraps to additional rows, scrollable

### Dimension Inputs
- Default: stone-300 border, empty field
- Focused: accent green border (ring-2)
- Filled valid: stone-300 border, value shown
- Error: red-600 border + error text below in red-600, 14pt
- Shape not yet selected: dimension fields hidden (only shape cards shown)

### Analysis Loading
- Loading: spinner + progress bar (animate-pulse), "Analyse abbrechen" button visible
- Error: AlertCircle red icon, error heading, "Erneut versuchen" + "Analyse abbrechen" buttons
- Budget soft-warning: amber InlineBanner above (on dimensions screen)
- Budget hard-stop: destructive InlineBanner + CTA disabled (on dimensions screen)

### Element Confirmation
- Default: all elements accepted (switch on), confidence badges shown
- Partially rejected: rejected rows have stone-400 text + switch off
- All rejected: all switches off, CTA still enabled (creates empty plan)
- Empty (0 elements): illustration + empty state copy + single CTA

### Plan View
- Default: SVG plan rendered with grid ON, all confirmed elements placed
- Grid off: grid lines hidden, elements and boundary remain
- Empty plan: boundary + grid only, no elements. Info text overlay: "Noch keine Elemente -- nutze den Editor zum Einzeichnen."

## Accessibility Contract

Inherited from Phase 2 baseline, with Phase 4 additions:

- All tappable elements: minimum 44px touch target (`min-h-[44px]`). Applies to shutter button (64px), gallery button, thumbnail grid items (80px), element rows, CTA buttons.
- Color is never the sole differentiator: confidence uses text label ("sicher"/"unsicher") in addition to color coding. Rejected elements use both gray color AND switch-off state.
- Step indicator dots: `accessibilityLabel="Schritt {n} von {total}"` on the dot container.
- Camera viewfinder: `accessibilityLabel="Kamerasucher -- Foto aufnehmen mit dem Button unten"`.
- Analysis loading: `accessibilityLabel="Garten wird analysiert"` + `accessibilityLiveRegion="polite"` for state transitions.
- Plan SVG: `accessibilityLabel="Schematischer Gartenplan mit {n} Elementen"` on the SVG container. Individual elements are decorative (no individual tap targets in Phase 4).
- Grid toggle button: `accessibilityLabel="Raster ein/aus"` on the icon button wrapping the Grid3x3 icon.
