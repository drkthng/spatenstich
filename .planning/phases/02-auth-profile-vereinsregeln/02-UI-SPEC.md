## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | NativeWind class | Usage |
|-------|-------|-----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, badge padding, rule-toggle spacing |
| sm | 8px | p-2 / gap-2 | Compact element spacing, input leading icon gap |
| md | 16px | p-4 / gap-4 | Default screen padding, form field spacing |
| lg | 24px | p-6 / gap-6 | Card padding, section header below |
| xl | 32px | p-8 / gap-8 | Section separation on profile screen |
| 2xl | 48px | p-12 / gap-12 | Auth-choice screen vertical centering space |
| 3xl | 64px | p-16 / gap-16 | Logo / illustration vertical offset on auth entry screen |

Exceptions:
- Touch targets: minimum 44px height on all tappable elements (native accessibility requirement). Use `min-h-[44px]` on buttons and list-row items.
- BKleingG rule rows: 48px minimum height (`min-h-12`) to ensure tappable toggle affordance.
- Inline banner: 4px left border accent stripe — this is decorative, not a spacing token.

## Color

Garden theme: earth tones with a natural green accent. Designed for readability in outdoor light on iPhone.

| Role | Light mode value | Dark mode (NativeWind dark:) | Usage |
|------|-----------------|------------------------------|-------|
| Dominant (60%) | `#F9F7F4` — warm off-white (stone-50 equivalent) | `#1C1917` — stone-900 | All screen backgrounds, auth-choice background |
| Secondary (30%) | `#E7E5E4` — stone-200 | `#292524` — stone-800 | Cards (archetyp cards, rule list rows), bottom sheet backgrounds, nav bar |
| Accent (10%) | `#4A7C59` — custom garden green | `#6BAA7E` — lightened for dark bg | Reserved-for list below |
| Destructive | `#DC2626` — red-600 | `#EF4444` — red-500 | Delete rule button, delete account action only |
| Semantic: Warning | `#D97706` — amber-600 | `#F59E0B` — amber-500 | BKleingG 1/3-Ampel amber state, inline banner for missing profile data |
| Semantic: Success | `#15803D` — green-700 | `#22C55E` — green-500 | BKleingG 1/3-Ampel green state, rule confirmed checkmark |
| Semantic: Neutral | `#78716C` — stone-500 | `#A8A29E` — stone-400 | BKleingG 1/3 "Plan noch nicht vorhanden" state, disabled rule toggles |

**Accent reserved for:**
- Primary CTA button background (Weiter, Speichern, Account erstellen, Lokal starten)
- Active Archetyp card border + checkmark
- Active toggle track for accepted Vereinsregeln
- Inline banner left-border stripe (warning banners use amber, not accent)

**60/30/10 split note:** Dominant covers all screen backgrounds. Secondary covers the rule list rows, archetyp cards, and profile section containers. Accent appears only on the specific interactive elements listed above — never on decorative elements, never as a background wash.

## Interaction Contract

### Auth-Wahl Screen (AUTH-05, CONTEXT.md Specifics)
- Two `AuthChoiceCard` components stacked vertically with 16px gap
- App name "Spatenstich" as Display text (28pt semibold) above cards
- One-line sub-caption below app name: "Dein digitaler Kleingarten-Assistent" (14pt, stone-500)
- Haftungsausschluss (NFR-07): shown as a collapsible note below the two cards. Default collapsed. Label: "Rechtlicher Hinweis" with chevron-down icon. When expanded shows: "Die App gibt Empfehlungen ohne Gewähr. BKleingG-Compliance liegt in der Verantwortung des Nutzers." This placement is Claude's discretion call — it does not block the CTA, it is accessible but not intrusive.
- Card tap → immediate navigation, no confirmation

### PDF-Extraction Loading Screen (D-07, Claude's Discretion)
- Full-screen overlay on top of upload screen (not a modal sheet)
- Center-aligned: animated plant-growth icon (Lottie or simple CSS animation via Reanimated) + heading "Regeln werden extrahiert…" (20pt semibold) + sub-text "Das kann 10–30 Sekunden dauern." (14pt, stone-500)
- Progress bar (indeterminate): accent green, full width, positioned below sub-text
- Cancel button: text-only ("Abbrechen"), 14pt, stone-500, at bottom of screen — tapping cancels the AbortController and returns to the upload entry screen with no error state
- On timeout (55s client-side): overlay transitions to error state — icon changes to alert-circle, heading changes to error copy (see Copywriting Contract), retry button appears

### Inline Banner (D-04, Claude's Discretion)
- Fixed height: auto (text wraps), minimum 52px
- Layout: 4px left border (amber for warnings) + 12px left padding + icon (16px, amber) + text column (message + action link) + dismiss X (right-aligned, 44px touch target)
- Dismiss: tapping X stores a dismissed flag in local state only — banner re-appears on next cold start if data is still missing. No permanent dismiss.
- Action link: tapping the banner body (not just the link) navigates to the relevant setting screen
- Shown: on Profil-Übersicht screen per missing data item. Not shown if data is already set.

### Local-Mode PDF-Upload Redirect (Claude's Discretion, Pitfall 4)
- On the Vereinsregeln-Einstieg screen, local-mode users see the PDF-Upload option as visually disabled: grayed card with lock icon (stone-400 text and border)
- Tapping the disabled card shows an inline info message beneath it (not a modal): "PDF-Upload erfordert einen Account. Mit einem Account kannst du auch deine Daten sichern." + a secondary CTA "Account erstellen" (text button, accent color)
- The Checklisten option below is fully active and the recommended path

### Vereinsregeln Bestätigung (D-08)
- BKleingG Grundregeln rendered first as a labeled group header "Gesetzliche Grundregeln (BKleingG)" with all rows in stone-400 (non-toggleable). Lock icon replaces switch.
- User-extracted rules below, labeled "Deine Vereinsregeln". Each row has Switch (default on = accepted) + edit icon (opens inline text input in the row, expanding the row height) + swipe-to-delete gesture (destructive, red background with trash icon)
- "Speichern"-button fixed at bottom, full width, accent background — disabled until user has scrolled past all BKleingG rows (ensures awareness, not compliance enforcement)
- Empty extracted list state: show "Keine Regeln erkannt. Du kannst sie manuell hinzufügen." + "Zur Checkliste" button

### BKleingG 1/3-Ampel (D-10)
- TrafficLightBadge: stone/neutral state with text "Plan noch nicht vorhanden"
- Tapping it: no navigation in Phase 2. Shows a tooltip/popover: "Die Nutz/Zier-Berechnung steht bereit, sobald du deinen Gartenplan in Phase 4 angelegt hast."

## States per Screen

### Input fields
- Default: stone-300 border
- Focused: accent green border (ring-2, accent color)
- Error: red-600 border + error message below in red-600, 14pt
- Disabled: stone-200 background + stone-400 text (BKleingG fields)
- Success (PLZ): stone-300 border + Klimazone badge appears below field immediately

### Buttons (Primary)
- Default: accent green background + white text
- Pressed: accent green darkened 10% (`active:opacity-80`)
- Disabled: stone-300 background + stone-400 text (no interaction)
- Loading: spinner replaces label, same background

### Archetyp Cards
- Default: secondary background + stone-400 border
- Selected: accent border (2px) + accent checkmark icon top-right + slightly elevated shadow
- Hover/pressed: stone-100 background flash

### Rule rows (Vereinsregeln)
- BKleingG row: stone-400 text, stone-300 toggle (locked), lock icon (14px, stone-400)
- User rule default: full opacity, switch on (accepted), edit and delete affordances visible
- User rule rejected (switch off): stone-400 text, switch off (stone-300 track)
- Swipe active: red-600 background, white trash icon revealed at 80% swipe

## Accessibility Contract

- All tappable elements: minimum 44px touch target (`min-h-[44px]`)
- Color is never the sole differentiator: BKleingG lock icon supplements the gray color on disabled rows
- Labels on all form inputs (not placeholder-only)
- Error messages announced via `accessibilityLiveRegion="polite"` on error text
- Loading states: `accessibilityLabel="Extrahiere Vereinsregeln"` on extraction loader
- BKleingG Ampel: `accessibilityLabel` includes the full state description (e.g. "BKleingG-Status: Plan noch nicht vorhanden")
