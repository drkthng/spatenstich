|----------|-------------|
| Client → Supabase Auth API | Untrusted password input crosses here; Supabase enforces password policy |
| Client → Supabase Postgres (via PostgREST) | RLS is the only barrier between users; service_role key MUST never reach client |
| Client device → SecureStore | Encrypted-at-rest store for AES key + local UUID; OS handles biometric/passcode gate |
| Client device → AsyncStorage | Plaintext store for ciphertext-only Supabase session blob; safe because key lives in SecureStore |
